# Documentation Writer Skill

## Overview

This skill helps you create clear, comprehensive documentation including README files, API docs, code comments, and technical guides.

## Activation Triggers

This skill activates when you:
- Need to write a README
- Want to document an API
- Need code documentation (docstrings, JSDoc)
- Want to explain complex code
- Need to create user guides

Example prompts:
- "Write a README for this project"
- "Document this API endpoint"
- "Add JSDoc comments to these functions"
- "Create documentation for this module"
- "Write a getting started guide"

## Documentation Types

- **README Files**: Project overviews and setup guides
- **API Documentation**: Endpoint and method documentation
- **Code Documentation**: Inline comments and docstrings
- **Architecture Docs**: System design explanations

## Supported Formats

- **JavaScript/TypeScript**: JSDoc
- **Python**: Google-style docstrings
- **Go**: Go doc comments
- **Markdown**: README, guides, wikis

## Example Usage

### Example 1: README Generation

**User**: "Create a README for this project"

**Claude**: Analyzes the project structure, dependencies, and code to generate a comprehensive README with installation, usage, and configuration sections.

### Example 2: API Documentation

**User**: "Document these REST endpoints"

**Claude**: Creates API documentation with endpoint descriptions, request/response formats, parameters, and example curl commands.

### Example 3: Code Comments

**User**: "Add documentation to this module"

**Claude**: Adds appropriate docstrings/comments to functions, classes, and complex logic using the language's standard documentation format.
